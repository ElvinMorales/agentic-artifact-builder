# Agentic Artifact Builder v0.1.0

Release date: 2026-06-17

## What This Release Is

`v0.1.0` is the first public MVP milestone for Agentic Artifact Builder, a small browser-based builder that helps people explore and generate starter artifacts for agentic AI systems using a taxonomy-first approach.

The project is the interactive companion layer for the source taxonomy repository:

- Source taxonomy repo: `https://github.com/ElvinMorales/agentic-ai-artifact-taxonomy`
- Live GitHub Pages demo: `https://elvinmorales.github.io/agentic-artifact-builder/`

The builder keeps the stable 14 top-level taxonomy buckets intact and gives learners a concrete way to inspect artifact types, fill guided fields, preview generated starter files, and download public-safe examples.

## What Changed

- Added a taxonomy-aligned artifact catalog mapped to the stable 14 top-level buckets.
- Added a static browser builder served directly from `src/`.
- Added catalog browsing by artifact type, taxonomy bucket, and lifecycle stage.
- Added guided fields, learning notes, synthetic example loading, preview, copy, reset, and download behavior.
- Added specialized renderers for all current catalog artifact types.
- Kept a generic fallback renderer for future catalog entries.
- Added a synthetic docs assistant example bundle.
- Added a public-safety checklist for generated artifacts.
- Added catalog integrity and renderer smoke tests.
- Added CI validation through GitHub Actions.
- Added GitHub Pages deployment for the static app.
- Added contribution docs, issue templates, and a pull request template.

## What Is Included

This release includes starter support for artifacts across the current catalog:

- Agent manifests and role profiles.
- Operating principles.
- Skill modules.
- Tool specs and resource manifests.
- System or task prompts and interface schemas.
- Memory policies and state strategies.
- Plan records and handoff contracts.
- Guardrails and governance policies.
- Output schemas.
- Evaluation rubrics and observability notes.
- Runtime configuration notes.
- Iteration or changelog notes.

The generated examples are synthetic and generic. They are intended for learning, adaptation, and public review, not production deployment.

## What Is Intentionally Not Included

This release does not include:

- A production agent runtime.
- Hosted memory, trace, or workflow storage.
- Authentication, accounts, or user-specific persistence.
- Live tool execution or external system writes.
- Private workflow templates.
- Regulated data examples.
- Framework-specific app architecture.
- Package manager, dependency, or build tooling.
- A replacement for the source taxonomy repository.

## Public-Safety Posture

The repository is designed to stay public-safe. Examples must remain synthetic and generic.

Do not add secrets, credentials, account ids, private URLs, employer-specific workflows, proprietary content, regulated data, private logs, real traces, memory entries, live state snapshots, or production runtime data.

Before publishing generated artifacts or screenshots, review:

- `docs/public-safety-checklist.md`

## How To Try It Locally

No install or build step is required. From the repository root, run:

```bash
python -m http.server 8000 -d src --bind 127.0.0.1
```

Then open:

```text
http://127.0.0.1:8000
```

## Known Limitations

- The app is a static MVP and stores form state only in the current browser session.
- Generated starter files are educational scaffolds, not complete production policies or runtime configs.
- The catalog is intentionally small and should continue to track the source taxonomy.
- Protocol and framework mappings are examples or adapters, not taxonomy replacements.
- The live demo URL should be confirmed from the successful GitHub Pages deployment before publishing release notes.
